## Gap Analysis

| Technique | Status | Likely reason undetected | Risk |
|-----------|--------|--------------------------|------|
| T1057 Process Discovery | SILENT | No rule watching `ps` / `/proc` enumeration; commands touch neither monitored files nor auth logs | MED |
| T1033 System Owner/User Discovery | SILENT | `whoami` / `id` not audited; no command-execution rule | LOW |
| T1087 Account Discovery | SILENT | `cat /etc/passwd` reads a file but no FIM/read rule covers enumeration | MED |

**Root cause:** the host's Wazuh configuration alerts on file *modification* (FIM) and authentication events, but discovery commands are read-only shell activity that touches neither. Without command auditing (auditd exec rules) or content-based detection rules, the entire Discovery tactic is invisible.

Prior work (Tripwire rule 101014) already detects **T1082 System Information Discovery** via `hostname` — proving the approach works. The gap is that coverage stops at that single technique.

---

## Recommended Next Detections

1. **T1057 — Process Discovery:** Tripwire-style rule matching `ps`, `top`, and reads of `/proc/*/status` by non-root UIDs.
2. **T1087 — Account Discovery:** rule on reads/enumeration of `/etc/passwd` and `/etc/group`.
3. **T1033 — System Owner/User Discovery:** auditd exec rule on `whoami` / `id`.
4. **Enable command auditing:** add auditd exec rules so shell commands generate events Wazuh can key on — foundational for the entire Discovery/Execution coverage story.

---

## Methodology Notes (for future campaigns)

- **Set `Auto Close: auto close operation`** so each campaign self-terminates after one clean pass. This gives a discrete, scoreable window with an unambiguous end time (v0.1 had to be stopped manually).
- **Mark start/end timestamps** on the Caldera host (`date "+%H:%M:%S"`) and score Wazuh with an **absolute** time filter — "Last N hours" pulls in unrelated noise.
- **Keep the first run of any profile plain-text** to test true rule coverage before layering in obfuscation. Obfuscated re-runs then measure evasion resilience separately.
